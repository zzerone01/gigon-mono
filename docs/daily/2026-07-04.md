### 푸시 알림 (Expo Push)

- DB: `push_tokens` (기기당 1행, match_pins처럼 클라 grant 0 — API 전용).
  로컬+클라우드 적용, 타입 재생성
- API: `/api/push/register`(upsert) · `/api/push/unregister`(소유자만 삭제)
- 발송: `lib/server/push.ts` — Expo Push API, 100개 청크, DeviceNotRegistered
  토큰 자동 삭제, `PUSH_DISABLED=1` 킬스위치. 핸들러에서 `defer()`(next `after()`
  래퍼 — 테스트 등 요청 컨텍스트 밖에선 폴백)로 응답 후 발송
- 이벤트 5개: 지원(→사장님) · 매칭(→워커) · 채팅(→상대) · 매칭취소(→상대) ·
  노쇼(→워커)
- 모바일: `src/lib/notifications.ts` — 권한 요청 + 토큰 등록(boot 시),
  로그아웃 시 unregister. **Expo Go에선 원격 푸시 불가(SDK 53+)라 조용히 no-op** —
  실수신은 EAS 빌드에서만
- 검증: vitest **34/34** (푸시 레지스트리 3케이스 추가), 웹·모바일 타입체크·린트,
  expo export, 클라우드 db push, Vercel 프로덕션 배포 + 401 프로브

## 사용자 액션 필요 (스토어 제출 전)

1. **Google Play Console 개발자 계정** ($25 일회성, play.google.com/console) —
   신원 확인이 며칠 걸릴 수 있어 **가장 먼저 시작할 것**. 계정 주체(개인/조직) 결정
2. **Firebase 프로젝트** (FCM — Android 푸시 수신에 필수):
   console.firebase.google.com → 프로젝트 생성 → Android 앱 추가(패키지 `io.gigon.app`)
   → `google-services.json` 다운로드 → `apps/mobile/`에 저장 (gitignore 됨)
   → 프로젝트 설정 > 서비스 계정 > 새 비공개 키 생성(JSON)
   → `cd apps/mobile && npx eas-cli credentials` → Android → Push Notifications →
   FCM V1 서비스 계정 키 업로드
3. **Android용 Google Maps 키**: 웹 키는 referer 제한이라 네이티브에서 안 됨.
   Google Cloud Console에서 새 키 생성 → "Maps SDK for Android" 활성화 →
   Android 앱 제한(패키지 `io.gigon.app` + EAS 키스토어 SHA-1) →
   `npx eas-cli env:create --name GOOGLE_MAPS_ANDROID_API_KEY --value <키>` (또는 eas.json env)
4. 2·3 완료 후: `cd apps/mobile && npx eas-cli build -p android --profile production`
   → `eas submit -p android` (Play Console 연결)
5. Play Console 스토어 등록정보: 스크린샷, 개인정보처리방침 URL(https://app.gigon.io/privacy),
   데이터 보안 양식

## 배포/빌드 치트시트

```bash
# JS 변경만 (심사·재빌드 불필요):
cd apps/mobile && npx eas-cli update --channel preview   # or production
# 네이티브 변경(키 추가, SDK 업그레이드 등):
npx eas-cli build -p android --profile production
# 실기기 테스트용 APK:
npx eas-cli build -p android --profile preview
```
