## 오늘까지 구현 완료된 것

### 1. Supabase 백엔드 (`supabase/`)

- PRD v1.2 §4.1 상태머신 전체 스키마: `profiles` `gigs` `applications` `matches`
  `messages` `reviews` `disputes` `billable_events` `audit_log`(append-only)
  `invite_codes` `match_pins`
- 모든 상태 전이는 **SECURITY DEFINER RPC로만** 수행 (클라이언트가 상태머신 우회 불가):
  `post_gig` `apply_to_gig` `select_applicant`(→billable_event ₱0)
  `mark_arrived` `issue_pin`(bcrypt) `verify_pin`(3회 오류→60초 잠금)
  `report_no_show` `open_dispute` `post_review` `send_message`
  `redeem_invite` `complete_onboarding` `set_active_role`
- RLS 방침(오너 결정): **전 테이블 RLS ON**(어드바이저 경고 방지)이되 row rule을
  권한 레이어로 쓰지 않음 — 읽기는 `authenticated using (true)` 한 줄, 쓰기 정책 없음.
  PIN 해시는 grant 없는 `match_pins` 테이블로 격리
- Realtime publication: `matches` `applications` `messages` `gigs`
- 클라우드 프로젝트 **`uoeatrexoeghoigbxage`** (ap-northeast-1)에 마이그레이션 4개 + 시드 적용,
  전화 OTP 인증 활성화(테스트 번호), `supabase link` 완료

### 5. 트러블슈팅 기록 (재발 방지)

| 문제 | 해결 |
| --- | --- |
| PostgREST 404/permission denied | 테이블에 `authenticated` GRANT 필요 (마이그레이션 `…010000`) |
| `gen_salt` not found | pgcrypto는 `extensions` 스키마 → PIN 함수 `search_path = public, extensions` |
| 로컬 phone OTP `phone_provider_disabled` | 더미 Twilio provider 활성화 필요 (test_otp 번호는 실발송 안 함) |
| Colima에서 vector 컨테이너 실패 | `[analytics] enabled = false` |
| Next 16 middleware deprecated | `proxy.ts` + `export function proxy` |
| 127.0.0.1 dev HMR 차단 | `allowedDevOrigins: ["127.0.0.1"]` |
| pnpm+Metro 모듈 해석 실패 | `disableHierarchicalLookup` 제거, `@expo/metro-runtime` 직접 의존성 |
| `next build`를 dev 서버 실행 중 돌리면 `.next` 충돌 | 동시에 돌리지 말 것 |
