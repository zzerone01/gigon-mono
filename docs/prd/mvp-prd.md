## 1. 제품 개요

**GigOn**은 필리핀의 소상공인과 인근 노동자를 연결하는 **지도 기반(map-first) 초단기 알바
마켓플레이스**다. 청소·세탁·배달·잔심부름 같은 **1–3시간짜리 일**을, 걸어갈 수 있는 거리의
검증된 사람과 즉시 매칭한다.

> One-liner: *GigOn connects small businesses with trusted workers nearby,
> for the quick 1–3 hour jobs you need filled right now.*

- 도메인: **gigon.io** · 태그라인: **Your gig is on.**
- 시장: 필리핀 전역 프라이빗 베타 (한국식 비유: "당근알바 for PH")
- 형태: 양면 마켓플레이스 — 한 계정으로 워커↔사장님 역할 전환 가능

## 2. 문제 정의

필리핀의 단기 일자리 거래는 대부분 **Facebook 그룹/단톡방**에서 일어난다. 그 결과:

| 페인 포인트 | 내용 |
| --- | --- |
| 노쇼 | 안 나타나도 아무런 책임·기록이 없다 |
| 무검증 | 누굴 고용하는지, 누구 밑에서 일하는지 알 수 없다 |
| 지불 불확실 | 워커는 돈 떼일 위험, 사장님은 악성 지원자 위험 |
| 근접성 부재 | 한참 스크롤했는데 도시 반대편 사람 |

## 3. 타겟 사용자

- **사장님(Business)** — 스파·식당·바·상점 등 소상공인. 갑작스런 러시나 결원을
  **오늘, 지금** 메워야 하는 상황.
- **워커(Worker)** — 자투리 시간에 걸어갈 수 있는 거리에서 현금 일당을 벌고 싶은 누구나.

## 4. 제품 원칙 (불변 4원칙)

1. **임금 미개입** — 급여는 현장에서 현금 100%. GigOn은 돈을 만지지 않는다.
2. **노동자 무과금** — 워커에게는 영원히 무료.
3. **PIN 상호 완료** — 일이 끝나면 사장님이 발급한 4자리 PIN을 워커가 입력해야 거래 완료.
   리뷰는 PIN 완료된 매칭에서만 가능 → 리뷰 신뢰성의 근간.
4. **파일럿 완전 무료** — 매칭 시 **₱0짜리 billable_event만 기록**(추후 과금 대비 데이터).
   파일럿 이후 사장님 대상 소액 매칭 수수료 도입 예정.

## 5. 제품 구성 (3개 접점, 하나의 계정·데이터)

```
gigon.io        랜딩 + 웨이트리스트 (수요 검증·유입)
app.gigon.io    서비스 웹앱 — 반응형 (모바일 웹 + 데스크톱 전용 레이아웃)
모바일 앱        Expo/React Native — Android·iOS 동일 코드베이스
```

- 웹과 모바일이 **같은 Supabase 계정·데이터**를 공유 → 웹에서 올린 공고를 앱에서 이어서 관리.
- 디자인은 Claude Design 3벌(`GigOn App`/`GigOn Web Mobile`/`GigOn Web`)을 1:1 구현.
- 지도: **Google Maps** (필리핀 사용자 친숙도 기준 선택).

## 6. 핵심 사용자 여정 (구현된 상태머신)

### 6.1 워커

```
탐색(리스트/지도, 거리순) → 1탭 지원(APPLIED) → 사장님 선택 시 MATCHED
→ 현장 도착 "I've arrived"(IN_PROGRESS) → 현금 수령 후 PIN 입력(COMPLETED)
→ 리뷰 작성(RATED)
```

### 6.2 사장님 — 파일럿 초대제 (초대코드 필요)

```
공고 등록(무료, 24h 유효) → 실시간 지원자 비교(거리·평점·완료수·노쇼/취소 이력)
→ 매칭 확정(billable_event ₱0 기록) → 워커 도착 알림 → 일 종료 시 PIN 발급
→ 워커가 PIN 입력하면 COMPLETED → 리뷰
```

### 6.3 예외 플로우 (전부 구현됨)

| 플로우 | 규칙 |
| --- | --- |
| **매칭 취소** | MATCHED 상태에서만, 양쪽 모두 가능. 사유 선택 필수. 취소자 프로필에 `cancel_count` 자동 기록. 워커 취소 → 공고 재오픈 / 사장님 취소 → 공고 종료 |
| **공고 철회** | 매칭 전(POSTED)만, 페널티 없음. 대기 지원자는 자동 거절 처리 |
| **노쇼 신고** | 사장님이 MATCHED 상태에서 신고 → 워커 프로필에 `no_show_count` 기록, 공고 재오픈 |
| **분쟁** | IN_PROGRESS/COMPLETED에서 양쪽 모두 가능. 티켓 번호 발급(D-10xx), 24시간 내 검토 안내 |
| **공고 자동 만료** | 등록 24시간 경과 시 자동 EXPIRED (DB pg_cron 10분 주기 + 앱 피드 로드 시 스윕) |

### 6.4 실시간성

지원·매칭·취소·노쇼·채팅이 상대방 화면에 **실시간 토스트 + 자동 갱신**으로 반영된다
(Supabase Realtime). 앱이 꺼져 있으면 푸시 알림(§7.8).

## 7. 기능 명세

### 7.1 온보딩 · 인증

- **전화번호 + SMS OTP** 로그인 (Supabase Auth). 비밀번호 없음.
- 온보딩: 전화번호 → OTP → 역할 선택(이름·상호명·초대코드) → 위치 설정.
- 계정 1개 = 역할 전환 가능(워커 ↔ 사장님). 프로필에서 즉시 전환.
- **사장님 진입 장벽**: 파일럿 초대코드(현재 `MACTAN-30`)를 인증해야 공고 등록 가능
  (invite-verified 배지).
- 실 SMS: 필리핀 로컬 사업자 **Semaphore** 연동 완료(~₱1.12/OTP, Twilio 대비 1/15 비용).
  ⏳ Semaphore 계정 승인·Sender Name·크레딧 충전 대기 — 그 전까지는 테스트 번호만 로그인 가능.

### 7.2 탐색 (워커)

- 공고 피드: **리스트 ↔ 지도 토글**, 거리순 정렬, 카테고리·급여·시간·거리 표시.
- 지도: Google Maps에 가격 핀 + 내 위치. 거리는 실좌표 기반 계산.
- 공고 상세: 업체 정보·평점·위치·급여·시간·설명 + 1탭 지원.

### 7.3 공고 (사장님)

- 등록 무료. 필드: 카테고리(**Cleaning / Laundry / Delivery / Errands**), 제목·설명,
  급여(₱), 시간대, 위치(지도 핀), 인원(slots).
- 유효기간 24시간 — 만료 시 자동 EXPIRED.
- 상태: `POSTED → MATCHED → COMPLETED | CLOSED / CANCELLED / EXPIRED`.

### 7.4 지원 · 매칭

- 워커: **1탭 지원** (공고당 1회, 본인 공고 지원 불가).
- 사장님: 지원자 카드 실시간 수신 — **거리·평점·완료 건수·노쇼/취소 이력**을 한눈에 비교.
- 매칭 확정 시: 다른 지원자 자동 거절, `billable_event(₱0)` 기록, 양쪽에 알림.
- 동시성 안전: 서버에서 조건부 UPDATE 레이스 가드 (이중 매칭 불가).

### 7.5 진행 · 완료 (PIN)

- 워커 도착 → "I've arrived" → IN_PROGRESS, 사장님에게 알림.
- 일 종료 → 사장님이 PIN 발급(서버가 무작위 4자리 생성, **bcrypt 해시만 저장**) →
  현금 지급 → 워커가 자기 폰에 PIN 입력 → COMPLETED.
- PIN 보안: 3회 오답 시 서버에서 60초 잠금. 해시는 클라이언트가 읽을 수 없는
  별도 테이블에 격리.

### 7.6 리뷰 · 평판

- **PIN 완료된 매칭에서만**, 참여자당 1건 (어뷰징 원천 차단).
- 별점 + 코멘트. 프로필에 평점 집계·완료 건수 자동 반영.
- 노쇼/취소 이력은 **자진 신고가 아니라 시스템이 자동 기록** — 평판을 조작할 수 없다.

### 7.7 채팅

- 매칭된 둘만, 인앱 1:1 채팅. 실시간(Realtime) 전송.
- **전화번호 비공개** — 연락은 인앱으로만.

### 7.8 푸시 알림 (모바일)

5개 이벤트에서 서버가 발송 (Expo Push / FCM V1):

`지원 접수(→사장님)` · `매칭 확정(→워커)` · `채팅 수신(→상대)` · `매칭 취소(→상대)` · `노쇼 처리(→워커)`

- 웹은 실시간 토스트로 동일 이벤트 수신. 알림 탭 시 해당 화면으로 라우팅.

## 8. 신뢰 · 안전 설계

| 장치 | 구현 |
| --- | --- |
| 상태머신 우회 불가 | 모든 상태 전이는 서버 API로만 — 클라이언트는 DB에 직접 쓸 수 없음 |
| PIN 보안 | bcrypt 해시 격리 저장 · 3회 오답 60초 잠금 · 평문은 발급 응답에만 존재 |
| 평판 무결성 | 노쇼/취소는 자동 기록 · 리뷰는 PIN 완료 매칭 한정 · 1인 1건 |
| 프라이버시 | 전화번호 비공개 · 채팅은 매칭된 둘만 · 인앱 한정 |
| 사장님 검증 | 파일럿 초대코드 필수 |
| 감사 추적 | 모든 상태 전이가 append-only `audit_log`에 기록 — 분쟁 판정 근거 |

## 9. 비즈니스 모델

- **현재(파일럿)**: 완전 무료. 단, 매칭 확정마다 `billable_event`를 ₱0으로 기록해
  **과금 시뮬레이션 데이터를 축적** — 요금제 설계의 근거가 된다.
- **Phase 1.5(파일럿 검증 후)**: 사장님 대상 소액 **매칭 성사 수수료**. 워커는 계속 무료.
- **임금에는 영원히 미개입** — 결제·정산·에스크로를 만들지 않는 것이 리스크·규제·신뢰
  측면의 의도된 전략.
- (탐색 중, 미구현) 리워드형 광고 등 보조 수익원.

## 10. 시스템 구성 (요약)

```
gigon-mono (turborepo + pnpm)
├── apps/web      gigon.io — 랜딩+웨이트리스트 (Next.js 16, Vercel)
├── apps/app      app.gigon.io — 서비스 웹앱 + API 서버 (Next.js 16, Vercel)
├── apps/mobile   Expo(React Native) — Android·iOS 단일 코드베이스, EAS 빌드/OTA
├── packages/ui   공유 디자인 시스템 (shadcn 63종 + GigOn 토큰 — 웹·모바일 공용)
└── supabase/     DB 스키마·마이그레이션 (Supabase 클라우드, 도쿄 리전)
```

- **백엔드**: Supabase (Postgres + 전화 OTP 인증 + Realtime) — 테이블 11개.
- **쓰기 API 17개**: Next.js Route Handlers — 모든 비즈니스 로직·권한 검증·감사 로그가
  서버 코드에 집중. vitest 통합 테스트 34케이스.
- **분석**: PostHog — 핵심 퍼널을 **서버사이드에서 캡처**하므로 웹·모바일이 계측 하나로
  커버됨. 랜딩 웨이트리스트→앱 사용까지 한 프로젝트에서 추적.
- **배포**: `main` 푸시 → Vercel 자동 배포. 모바일 JS 변경은 EAS Update(OTA)로
  스토어 심사 없이 배포 가능.

## 11. 지표 (KPI)

핵심 퍼널: **공고 → 지원 → 매칭 → 도착 → PIN 완료** (전 구간 이벤트 계측 완료)

- 워커 퍼널: `gig_applied → match_created → worker_arrived → match_completed → review_submitted`
- 사장님 퍼널: `gig_posted → match_created → pin_issued → match_completed`
- 랜딩 전환: `pageview → waitlist_joined`
- 보조 지표: 취소율·노쇼율·분쟁 발생률·리뷰 작성률 (이벤트 수집 중)

### 파일럿 오픈 전 잔여 (외부 대기 중심)

### 기능 백로그 (PRD 범위 내, 파일럿 후)

## 13. Out of Scope (의도된 미구현)

| 항목 | 이유 |
